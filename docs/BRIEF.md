# CareerOS Design System

---

# Design Philosophy

CareerOS should feel like professional software rather than a student project.

The interface should communicate confidence, clarity, and focus.

Every design decision should support productivity instead of competing for attention.

The application should feel modern in 2026 while remaining timeless.

# Things To Avoid

* Neon RGB aesthetics
* Crypto dashboard designs
* Glassmorphism everywhere
* Heavy gradients
* Overly animated interfaces
* Scroll hijacking
* Floating blobs
* Particle backgrounds
* Excessive shadows
* Empty decorative elements

Beauty should come from layout and typography, not visual effects.

# Typography

Primary Font

Fallback

Inter

Guidelines

* Large headings should be rare.
* Use typography hierarchy instead of colors.
* Let whitespace create emphasis.
* Never use more than three font weights on a page.

---

# Layout

Three-column desktop layout

Sidebar

↓

Content

↓

Utility Panel

Pages should never feel crowded.

Large spacing is preferred over squeezing information.

---

# Cards

Cards are the foundation of the interface.

Each card should answer exactly one question.

Examples

"What should I do today?"

"What's next?"

"How am I progressing?"

"What's changing?"

Cards should never become mini dashboards themselves.

---

# Widgets

Widgets are modular.

Every widget can appear in multiple pages.

Examples

* GitHub Widget
* LeetCode Widget
* Calendar Widget
* Project Widget
* Opportunity Widget
* Analytics Widget

Widgets should be independent and reusable.

---

# Navigation

Persistent Sidebar

Global Search

Command Palette

Breadcrumbs

Quick Actions

Navigation should require minimal clicks.

---

# Motion

Animations should support understanding.

Allowed

* Fade
* Scale
* Slide
* Card Lift
* Hover Feedback
* Skeleton Loading

Avoid

* Page transitions
* Long animations
* Bounce effects
* Overshooting
* Physics-based gimmicks

Target duration

150–250ms.

---

# Icons

Use Lucide Icons exclusively.

Icons should support labels rather than replace them.

Avoid mixing icon libraries.

---

# Charts

Charts should be readable first.

No unnecessary colors.

Use clean line charts.

Use subtle grid lines.

Prioritize trends over decoration.

---

# Responsiveness

Desktop First

Laptop

Tablet

Mobile

The experience should remain functional across all devices without hiding important information.

---

# Accessibility

Keyboard navigation

Visible focus states

Semantic HTML

High contrast

Readable font sizes

Screen reader support where possible

---

# User Experience Principles

The user should always know:

* What to do next.
* What changed recently.
* What deserves attention.
* How much progress has been made.
* What opportunities are approaching.

Every page should answer one of these questions.

---

# Overall Feeling

Opening CareerOS should feel like opening a premium piece of software that was built specifically for ambitious developers.

The interface should feel calm, polished, and trustworthy.

The application should encourage consistency rather than overwhelm the user.

Users should leave every session with a clearer understanding of what they need to do next.
